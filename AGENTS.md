## Project Overview

FreeCAD workbench for creating **Gridfinity+** baseplates and bins. Shapes are generated by
opening parameterized FreeCAD template files (`.FCStd`), writing values into their
`Spreadsheet` objects, recomputing, extracting the resulting `Part::Feature` shape, and
closing the template document. The extracted shape is placed into the user's active document
as a `Part::Feature`.

## Installation (Development)

Symlink (or copy) this repo into FreeCAD's Mod directory, then restart FreeCAD:

```
Windows: %APPDATA%\FreeCAD\Mod\gridfinity_plus_workbench  →  <this repo>
Linux:   ~/.FreeCAD/Mod/gridfinity_plus_workbench         →  <this repo>
```

## Architecture

```
init_gui.py → GridifintyPlusWorkbench
  └── Initialize() → registers CreateBaseplateCommand, CreateBinCommand

Command flow:
  Command.Activated()
    └── showTaskPanel()
          └── FreeCADGui.Control.showDialog(TaskPanel(command))
                └── TaskPanel button click → command.Create*()
                      └── template load → spreadsheet write → recompute → shape copy → close
```

## Key Conventions

- **No `print()`** — use `FreeCAD.Console.PrintMessage/PrintWarning/PrintError`.  
  See `.agents/skills/gfplus_logging/SKILL.md`.
- **No silent exceptions** — every `except` block must log before continuing.  
  See `.agents/rules/no_silent_exceptions.md`.
- **Template loading** follows a strict open-parameterize-extract-close pattern.  
  See `.agents/skills/gfplus_template_loading/SKILL.md`.
- **Command + TaskPanel pattern** must be followed for all new tools.  
  See `.agents/skills/gfplus_command_pattern/SKILL.md`.
- **Scope control** — change only what is asked.  
  See `.agents/rules/scopecontrol.md`.

## Running the Workbench

There are no automated tests. Launch FreeCAD with this workbench installed and exercise the
commands manually. The workbench creates objects in the active FreeCAD document.
